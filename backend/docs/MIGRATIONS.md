# Database Migrations Guide

## Pré-requisitos

1. **Supabase Project criado**
2. **Credenciais configuradas no `.env`**
3. **Python dependencies instaladas**: `pip install -r requirements.txt`

---

## Comandos Alembic

### Ver Status Atual

```bash
cd backend
alembic current
```

### Aplicar Migrations (Upgrade)

```bash
# Aplicar todas migrations pendentes
alembic upgrade head

# Aplicar próxima migration
alembic upgrade +1

# Aplicar até revisão específica
alembic upgrade <revision_id>
```

### Reverter Migrations (Downgrade)

```bash
# Reverter última migration
alembic downgrade -1

# Reverter todas migrations
alembic downgrade base

# Reverter até revisão específica
alembic downgrade <revision_id>
```

### Criar Nova Migration

```bash
# Autogenerate migration (detecta mudanças nos models)
alembic revision --autogenerate -m "Description of changes"

# Criar migration vazia (manual)
alembic revision -m "Description"
```

### Histórico

```bash
# Ver histórico de migrations
alembic history

# Ver histórico detalhado
alembic history --verbose
```

## Migration Inicial

A migration inicial (`001_initial`) cria as seguintes tabelas:

1. **users** - Usuários do sistema
2. **user_profiles** - Perfil estendido
3. **documents** - Documentos enviados
4. **ocr_results** - Resultados OCR
5. **tax_events** - Eventos tributários
6. **declarations** - Declarações IRPF
7. **validations** - Validações
8. **ai_interactions** - Interações IA
9. **audit_logs** - Logs de auditoria (LGPD)

---

## Aplicar Migration Inicial

### Passo 1: Configurar DATABASE_URL

Edite `.env` e configure:

```env
DATABASE_URL=postgresql+asyncpg://postgres:SUA_SENHA@db.xyz.supabase.co:5432/postgres
```

### Passo 2: Executar Migration

```bash
cd backend
alembic upgrade head
```

**Saída esperada:**
```
INFO  [alembic.runtime.migration] Running upgrade  -> 001_initial, Initial schema
```

### Passo 3: Verificar Schema

```bash
python scripts/verify_schema.py
```

**Saída esperada:**
```
✓ users
✓ user_profiles
✓ documents
✓ ocr_results
✓ tax_events
✓ declarations
✓ validations
✓ ai_interactions
✓ audit_logs

✓ ALL TABLES CREATED SUCCESSFULLY
```

## Resolver Problemas Comuns

### Erro: "relation already exists"

**Causa:** Tabelas já existem no banco.

**Solução:**
```bash
# Opção 1: Marcar migration como aplicada (sem executar)
alembic stamp head

# Opção 2: Dropar todas tabelas e recriar (CUIDADO: perde dados)
# No Supabase SQL Editor:
DROP SCHEMA public CASCADE;
CREATE SCHEMA public;
# Depois:
alembic upgrade head
```

### Erro: "Can't locate revision identified by..."

**Causa:** Alembic version table inconsistente.

**Solução:**
```bash
# Resetar alembic version
alembic stamp base
alembic upgrade head
```

### Erro: "No module named 'app'"

**Causa:** Python path incorreto.

**Solução:**
```bash
# Certifique-se de estar no diretório backend/
cd backend

# Ou adicione ao PYTHONPATH
export PYTHONPATH="${PYTHONPATH}:$(pwd)"  # Linux/Mac
$env:PYTHONPATH = "$(pwd);$env:PYTHONPATH"  # PowerShell
```

### Erro: "asyncpg.exceptions.InvalidPasswordError"

**Causa:** Senha incorreta no DATABASE_URL.

**Solução:**
- Verifique se a senha no `.env` está correta
- Verifique se substituiu `[YOUR-PASSWORD]` pela senha real

## Boas Práticas

### 1. Sempre criar backup antes de migration em produção

```bash
# PostgreSQL dump
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d_%H%M%S).sql
```

### 2. Testar migrations em ambiente de desenvolvimento primeiro

```bash
# Aplicar
alembic upgrade head

# Testar aplicação

# Se houver problema, reverter
alembic downgrade -1
```

### 3. Revisar migration autogenerated antes de aplicar

```bash
alembic revision --autogenerate -m "Add new column"

# IMPORTANTE: Revisar arquivo gerado em alembic/versions/
# Alembic pode não detectar todos os tipos de mudanças!
```

### 4. Nunca editar migrations já aplicadas

- Se migration já foi aplicada, criar nova migration para corrigir
- Não editar arquivos de migration existentes

### 5. Usar transações

- Alembic executa migrations em transação por padrão
- Se migration falhar, rollback automático

## Migration Workflow Recomendado

```
1. Desenvolver → Alterar models em app/shared/models/
2. Gerar migration → alembic revision --autogenerate -m "..."
3. Revisar → Verificar arquivo gerado em alembic/versions/
4. Testar local → alembic upgrade head
5. Verificar → python scripts/verify_schema.py
6. Commit → git add + git commit
7. Deploy → Executar em produção (com backup!)
```
